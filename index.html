Cours GIT

Qu'est-ce que GIT

GIT est un outil de versionning et il permet d'avoir une tracabilité et un historique de tous les fichiers du projet.

Il permet egalement de travailler en equipe: Envoyer nos modifications a nos collegues et recuperer les modifications des collegues

Dans la tracabilité des fichiers GIT nous donne toujours 3 infos: Qui, Quand, Pourquoi ?

Installer et configurer GIT

git config --global user.email 'iver91100@gmail.com'
git config --global user.name 'Nicolas'

Les commandes de base

Initialiser un projet:
git init
Voir le statut des fichiers
git status
Ajouter des fichiers
git add 'fichier'
Enregistrer (commit) des fichier (les fichiers add)
git commit
Un commit nécessite toujours un message
Commit avec un message :
git commit -m "message"
Commit avec add + message:
git commit -a -m "message"

Corection des fichiers

Voir les differents commit:
git log
Corriger les modifications sur un fichiers ( pas encore add):
git checkout 'fichiers'
Suprime un add:
git reset 'fichier'
git reset .
Supprimer le dernier commit:
git reset HEAD^
Supprimer les deux derniers commit:
git reset HEAD^^

La notion de branche

Une branche est une copie du projet, elle permet de developper en parralèlle de nouvelles fonctionnalités. Lorsqu'on considere les nouveautés opérationnelles, on peut les rappatrier sur la version principale (master)

Voir les branches:
git branch
Creer une branche:
git branch 'nom'
Se placer sur une branche:
git checkout 'nom'
Voir les différences entre deux branches:
git diff 'nom' ..master
Apporter les modif effectués sur une branche dans master
git checkout master
git merge 'nom'
Remplacer master par uen branche:
git checkout master
git fetch 'nom'
=git fetch va littéralement remplacer master par la branche. Il s'agit donc d'une grosse modification dans le projet (ex: version2)
